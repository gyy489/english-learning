# 每日英语 Web 应用

这个本地应用直接读写项目中的 `作文素材/按时间排序/`，不创建额外的数据格式。

页面支持浅色和暗色主题，首次打开时跟随系统设置，手动切换后会记住选择。
标题上方可以选择任意已有天数；正文、MP3 和生词区域会一起切换。
正文使用英文单词下方对应中文词义的逐词硬翻译，不再显示整句中文，用来帮助熟悉英文语序；冠词、代词、介词、连词和简单助动词等基础词不显示释义，只保留较难的名词、动词和形容词。鼠标移入一句正文时，该句所有词义会一起变清晰。
正文中的单词可单击查看 ECDICT 中英释义，双击则加入或移出当天生词。词典在本地运行，不需要 API Key。
网页和 VSCode 使用同一份 `作文素材/按时间排序/` Markdown 数据。VSCode 修改当前文章并保存后，网页会自动检查并同步最新内容；网页记录的生词也会直接写回同一个 Markdown 文件。

## 启动

在项目根目录运行：

```bash
uv run --with openai python english-learning-web/server.py
```

然后访问 <http://127.0.0.1:8787>。

只阅读文章、播放 MP3 和记录生词时，也可以直接运行：

```bash
python3 english-learning-web/server.py
```

网页加载后会登记当前浏览器页面；关闭网页时会通知后端。最后一个页面正常关闭后，后端会等待几秒并自动退出。页面静置、切到后台或电脑休眠不会触发超时退出；如果浏览器异常崩溃而没有发出关闭通知，后端会暂时保留，下一次打开 `/Applications/每日英语.app` 时直接复用，不会重复启动。

为了方便回忆，复习生词释义和正文中文默认显示得很淡并带有轻微模糊；鼠标移到中文解释上时会暂时恢复可读状态。

## 长期单词复习

页面顶部的“长期复习计划”会显示词库总数、活跃池、收件箱、到期词、长期维护池、归档数量，以及下一篇文章将使用的目标词。系统根据每日 Markdown 自动维护：

- `作文素材/单词复习/vocabulary.json`：机器可读的词库和间隔状态。
- `作文素材/单词复习/review-history.md`：方便查看的复习概览、习惯指标与历史。
- `作文素材/单词复习/README.md`：完整复习策略。

成功复习后采用 1、3、7、14、30、60 天的间隔；仍不会时回到次日。词汇经过“收件箱（120）→ 活跃学习池（60）→ 长期维护池（120）→ 归档”。每篇最多安排 15 个目标词，每次最多激活 2 个新词；有到期积压时暂停激活。这样词库可以增长，但固定复习负荷不会无限增长。页面同时显示最近 7 篇回忆成功率和平均生词量。

## 生成下一天

生成文章会优先使用项目 `.env` 中的 OpenAI API；没有 API Key 时，会自动寻找本机已登录的 Codex CLI，包括从 macOS App 启动时无法继承到 PATH 的 VS Code Codex。生成 MP3 需要本机可用的 `OPENAI_API_KEY`：从“应用程序”启动时会读取 `~/.zshrc` 中的 OpenAI 配置，手动启动时也可以使用项目根目录的 `.env`：

```text
OPENAI_API_KEY=你的_API_Key
```

默认使用低成本模型：

```text
OPENAI_TEXT_MODEL=gpt-4o-mini
OPENAI_TTS_MODEL=tts-1
```

`gpt-4o-mini` 用于文章和中文解释；`tts-1` 用于生成听力 MP3。两项都可以在 `.env` 中覆盖。

生成规则：

- 有未排入本篇的到期旧词：进入“到期词清理”模式，不主动加入新目标词。
- 总目标词不超过 7 个：结合新题库生成约 36-40 句，但额外新目标词仍受系统给出的 0-2 个额度限制。
- 总目标词为 8-11 个：生成至少 30 句，额外新目标词仍受 0-2 个额度限制。
- 总目标词为 12-15 个：生成至少 30 句，不主动加入新目标词。

生成的 Markdown 和 MP3 仍保存在新的每日文件夹中，因此可以继续在 VS Code 中直接使用。
