# 每日英语 Web 应用

这个本地应用直接读写项目中的 `作文素材/按时间排序/`，不创建额外的数据格式。

页面支持浅色和暗色主题，首次打开时跟随系统设置，手动切换后会记住选择。
标题上方可以选择任意已有天数；正文、MP3 和生词区域会一起切换。
正文中的单词可单击查看 ECDICT 中英释义，双击则加入或移出当天生词。词典在本地运行，不需要 API Key。
网页和 VSCode 使用同一份 `作文素材/按时间排序/` Markdown 数据。VSCode 修改当前文章并保存后，网页会自动检查并同步最新内容；网页记录的生词也会直接写回同一个 Markdown 文件。

## 启动

在项目根目录运行：

```bash
uv run --with openai python english-learning-web/server.py
```

然后访问 <http://127.0.0.1:8787>。

只阅读文章、播放 MP3 和记录生词时，也可以直接运行：

```bash
python3 english-learning-web/server.py
```

网页加载后会登记当前浏览器页面；关闭网页时会通知后端。最后一个页面关闭后，后端会等待几秒并自动退出；如果浏览器异常退出，心跳超时后也会自动清理。重新打开 `/Applications/每日英语.app` 会重新启动服务。

为了方便回忆，复习生词释义和正文中文默认显示得很淡并带有轻微模糊；鼠标移到中文解释上时会暂时恢复可读状态。

## 生成下一天

生成文章会优先使用项目 `.env` 中的 OpenAI API；没有 API Key 时，会自动使用本机已登录的 Codex CLI。生成 MP3 仍需要在项目根目录的 `.env` 中配置：

```text
OPENAI_API_KEY=你的_API_Key
```

默认使用低成本模型：

```text
OPENAI_TEXT_MODEL=gpt-4o-mini
OPENAI_TTS_MODEL=tts-1
```

`gpt-4o-mini` 用于文章和中文解释；`tts-1` 用于生成听力 MP3。两项都可以在 `.env` 中覆盖。

生成规则：

- 前一天生词超过 15 个：不读取题库，只根据旧生词生成约 25-30 句纯复习文章。
- 前一天生词为 0-5 个：旧生词结合新题库文章，生成约 40 句。
- 前一天生词为 6-15 个：旧生词结合新题库文章，生成至少 30 句。

生成的 Markdown 和 MP3 仍保存在新的每日文件夹中，因此可以继续在 VS Code 中直接使用。
