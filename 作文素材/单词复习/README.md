# 单词间隔复习策略

## 目标

这套系统解决两个问题：旧词长期不出现容易遗忘；一次把所有到期词塞进文章会破坏文章质量。每日文章最多安排 15 个目标复习词，并优先保证连贯、自然和适合朗读。

## 复习间隔

单词成功回忆后，复习间隔依次延长：

```text
1 天 → 3 天 → 7 天 → 14 天 → 30 天 → 60 天
```

如果复习时仍然不会，间隔重置为 1 天；连续成功后逐步进入“基本熟悉”和“已掌握”。已掌握不代表永久删除，仍会按较长间隔再次出现。

## 如何判断记住或遗忘

系统以每日 Markdown 为事实来源：

- 当前文章 `## 复习生词` 中的词，是本篇计划复习的词。
- 阅读后仍然不会的词，继续写入底部 `生单词:`，系统记录为“仍不会”。
- 本篇计划复习、但没有再次写入 `生单词:` 的词，在生成下一天内容时记录为“记住”。
- 正文里新遇到并加入 `生单词:` 的词，加入长期词库并在次日优先复习。
- 一个历史词在普通正文中再次被标记，记录为“旧词再次遗忘”，并重置到次日复习。

## 每日文章负荷

每篇文章的目标复习词上限是 15 个：

1. 先放当前文章底部的生词，最多 15 个。
2. 当前生词为 0-5 个时，最多补 5 个到期旧词。
3. 当前生词为 6-10 个时，最多补 4 个到期旧词。
4. 当前生词达到 11 个时，不再补到期旧词，也不主动引入新目标词。
5. 没有进入文章的到期旧词不会丢失，而是按逾期时间和遗忘次数排序，顺延到后续文章。
6. 只要仍有到期旧词在排队，文章就暂停引入新的目标词，先逐步清理复习积压。

没有复习积压时，新词负荷才随复习压力调整：目标词不超过 7 个时可加入 3-5 个新 IELTS 词；8-11 个时只加入 1-2 个；12 个以上不主动加入新目标词。

## 记录文件

- `vocabulary.json`：机器可读的词库、间隔、复习次数、遗忘次数和下一次复习天数。
- `review-history.md`：方便人工阅读的概览、复习习惯指标、下一篇计划、完整词库和最近复习历史。

习惯指标包括已完成学习篇数、累计和最近 7 篇的回忆成功率、平均每篇标记生词数。它们全部由每日 Markdown 推导，不需要额外打卡。

这两个文件由 Web App 根据每日文章自动重建。日常学习时只需要继续维护每日文章底部的 `生单词:`，不要手工修改 JSON。
