# 单词间隔复习策略

## 目标

这套系统解决两个问题：旧词长期不出现容易遗忘；持续加入生词会让待复习数量无限增长。系统不把“记录过的所有词”都当成“今天必须复习的词”，而是使用有容量上限的分层词池；每日文章最多安排 15 个目标词。

## 复习间隔

单词成功回忆后，复习间隔依次延长：

```text
1 天 → 3 天 → 7 天 → 14 天 → 30 天 → 60 天
```

如果复习时仍然不会，间隔重置为 1 天；连续成功后逐步进入“基本熟悉”。通过一次完整的 60 天间隔回忆后进入“已掌握”并归档；归档不删除记录，日后再次遇到且仍不会时会重新激活。

## 如何判断记住或遗忘

系统以每日 Markdown 为事实来源：

- 当前文章 `## 复习生词` 中的词，是本篇计划复习的词。
- 阅读后仍然不会的词，继续写入底部 `生单词:`，系统记录为“仍不会”。
- 本篇计划复习、但没有再次写入 `生单词:` 的词，在生成下一天内容时记录为“记住”。
- 正文里新遇到并加入 `生单词:` 的词，先进入收件箱；系统有余量时每篇最多激活 2 个。
- 一个历史词在普通正文中再次被标记，记录为“旧词再次遗忘”，并重置到次日复习。

## 四层词池

1. **收件箱（最多 120 个）**：刚发现、尚未正式进入间隔复习的词。收件箱满后，较晚且一直未被激活的词进入归档，不占用每日复习容量。
2. **活跃学习池（最多 60 个）**：正在进行 1、3、7 天短间隔巩固的词。只有这里的词会形成主要复习负荷。
3. **长期维护池（最多 120 个）**：已经基本熟悉、按 14、30、60 天间隔维护的词；每天最多抽取 2 个。
4. **归档**：通过一次完整的 60 天间隔回忆，或超出前面词池容量的词，不参加固定日程。以后在正文里再次被标记时，可以重新进入活跃学习池。

词库总数可以继续增长，但“活跃学习池 + 长期维护池”都有硬上限，因此必须定期复习的数量不会无限增长。归档不是删除，历史、释义和成绩仍保留在 JSON 与 Markdown 中。

## 每日文章负荷

每篇文章的目标词上限固定为 15 个，按下面的顺序分配：

1. 仍不会或重新遗忘的词优先，最多 10 个。
2. 长期维护池每天最多抽取 2 个，避免老词永久消失。
3. 剩余位置交给活跃学习池的到期词，整篇合计不超过 15 个。
4. 只有当天全部到期词都能排入文章、活跃池还有空位时，才从收件箱激活新词，每篇最多 2 个。
5. 只要有到期词顺延，就暂停激活新词；只要收件箱不空，文章也不主动制造额外的新目标词。

这个阀门使系统具备负反馈：复习压力升高 → 新词入口自动关闭；旧词逐渐掌握 → 活跃池释放位置 → 每次只恢复最多 2 个新词。即使某天标记 30 个生词，下一篇也不会因此塞入 30 个目标词。

## 记录文件

- `vocabulary.json`：机器可读的词库、间隔、复习次数、遗忘次数和下一次复习天数。
- `review-history.md`：方便人工阅读的概览、复习习惯指标、下一篇计划、完整词库和最近复习历史。

习惯指标包括已完成学习篇数、累计和最近 7 篇的回忆成功率、平均每篇标记生词数。它们全部由每日 Markdown 推导，不需要额外打卡。

这两个文件由 Web App 根据每日文章自动重建。日常学习时只需要继续维护每日文章底部的 `生单词:`，不要手工修改 JSON。
